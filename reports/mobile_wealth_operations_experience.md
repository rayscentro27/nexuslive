## Mobile Experience Design Principles

```
MOBILE-FIRST DESIGN RULES
══════════════════════════════════════════════════════════════════════════════════

  THUMB ZONES:
  ┌───────────────────────────────────┐
  │  HARD REACH                       │
  │  ─────────────────────────────    │
  │  [Status indicators / read-only]  │
  │  [Metric cards / numbers]         │
  │                                   │
  │  EASY REACH ────────────────────  │
  │  [Primary action buttons]         │
  │  [Navigation tabs]                │
  │  [Confirm / skip buttons]         │
  │  ─────────────────────────────    │
  │  BOTTOM NAV (always reachable)    │
  │  [FOREX] [OPTIONS] [CRYPTO] [AI]  │
  └───────────────────────────────────┘

  INFORMATION HIERARCHY (per card):
  1. Status (one word: WATCHING / READY / DONE / ACTIVE)
  2. Key number (price, P/L, streak)
  3. Action (what to do next)
  4. Context (secondary detail, smaller text)

  MAX INFORMATION PER SCREEN: 5–7 data points
  NO SCROLLING for primary status — above the fold always

  INTERACTION TIME TARGET:
  Morning check: 90 seconds
  Setup alert review: 30 seconds
  Evening recap: 2 minutes
  Full review: 5 minutes (weekly only)
```
